La virtualisation

Cette page présente le principe de la virtualisation, ses différences avec l’émulation et la simulation, ainsi que les types de virtualisation selon différents environnements logiciels.

Définition de la virtualisation

La virtualisation est une technologie qui permet de faire fonctionner plusieurs systèmes d’exploitation sur une seule machine physique. Elle repose sur un logiciel appelé hyperviseur, chargé de répartir les ressources matérielles entre plusieurs machines virtuelles.

Différence entre émulateur, simulateur et virtualisation

Émulateur

Un émulateur reproduit le fonctionnement d’un autre système matériel ou logiciel afin d’exécuter des programmes conçus pour une architecture différente.

Simulateur

Un simulateur imite le comportement d’un système sans reproduire exactement son matériel. Il est principalement utilisé pour l’apprentissage et les tests.

Outil de virtualisation

Un outil de virtualisation permet d’exécuter plusieurs systèmes d’exploitation réels sur une même machine en utilisant directement le matériel, ce qui offre de bonnes performances.

Types de virtualisation selon l’environnement

VirtualBox

VirtualBox utilise une virtualisation de type 2, où l’hyperviseur est installé sur un système d’exploitation hôte. Les machines virtuelles s’exécutent donc au-dessus de celui-ci, ce qui rend l’outil simple à utiliser pour la formation et les tests.

VMware Workstation

VMware Workstation repose également sur une virtualisation de type 2. Il fonctionne sur un système d’exploitation hôte et propose des fonctionnalités avancées, ce qui le rend adapté à un usage plus professionnel.

Stratus

Stratus utilise une virtualisation de type 1, dans laquelle l’hyperviseur est installé directement sur le matériel. Ce type de virtualisation offre de meilleures performances et une grande fiabilité pour les environnements critiques.

Forme et lieu de sauvegarde de la machine virtuelle

Les données d’une machine virtuelle sont principalement sauvegardées sous forme de fichiers stockés sur le disque de la machine hôte. Le disque virtuel contient le système d’exploitation, les logiciels et les données, tandis que des fichiers de configuration décrivent les ressources matérielles attribuées.

La mémoire vive (RAM) n’est pas sauvegardée de manière permanente. Elle peut cependant être enregistrée temporairement lors de la création d’un snapshot ou lorsque la machine virtuelle est mise en pause.

Les sauvegardes sont généralement stockées sur le disque dur ou le SSD de la machine hôte, et peuvent également être copiées sur un support externe ou un stockage réseau dans un contexte professionnel.
